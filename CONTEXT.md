# AdGuardHome DNS 延时分析

分析和诊断 AdGuardHome DNS 查询延时的工具和文档。

## 语言

**DNS 查询延时分析**:
对 AdGuardHome 查询日志进行统计，按域名维度聚合延时指标，找出长尾慢查询进行进一步诊断。
_Avoid_: DNS 速度测试、网络测速

**分析时间窗口**:
被分析数据的时间范围。当前目标为最近 24 小时。

**长尾慢域名**:
在延时分布中处于 P95/P99 以上的域名，需要进一步分析其上游路由、缓存策略或上游服务器健康状态。

**缓存命中记录**:
在统计报告中，缓存命中（cached=true）的查询单独统计，不与真实上游查询混算。统计报告分两列展示：`uncached`（反映上游真实性能）和 `all`（反映用户实际体验）。慢查询分析中缓存命中率是关键辅助指标——缓存命中率高即使偶有慢查询也无大碍，缓存命中率低且慢查询率高才是毒瘤。
_Avoid_: 排除、忽略、扔掉缓存数据

**慢查询分级**:
- **慢**: >500ms — 黄色标记，表示一般延迟
- **严重**: >1s — 橙色标记，表示用户可感知的卡顿
- **超时**: >3s — 红色标记，基本算超时
每个域名展示 `慢查询率`（>500ms 占比）和 `严重率`（>1s 占比），配合缓存命中率判断实际影响。

**查询类型分析**:
按 `question.type`（A/AAAA/PTR/MX/TXT/CNAME 等）分类统计。表格中显示类型分布（如 `A: 80%, AAAA: 15%, PTR: 5%`），支持按类型筛选。

**分步深度分析**:
慢域名分析采用两步法——第一步按域名聚合得出 Top N 慢域名总览（含 P20/P50/P80/P95/P99/Max/Avg/Min 及查询次数、缓存命中率、慢查询率、查询类型分布）；第二步针对长尾域名展开其上游服务器明细，定位具体是哪个上游拖慢了该域名的解析。

**配置与鉴权**:
AdGuardHome 连接信息（URL、用户名、密码）可通过网页配置面板设置，存储在后端进程内存中。前端同时缓存到 localStorage 以便下回自动填充。`.env` 文件可做启动时的默认配置。两种方式互不冲突。

**微服务架构**:
Vite + React SPA（前端）+ Fastify + TypeScript（后端）。前端负责数据展示和可视化；后端负责转发 AdGuardHome API、处理鉴权、缓存查询日志、数据预处理。

**数据获取策略**:
后端缓存计算结果（最近 24 小时），前端加载时直接读取缓存，手动点"刷新"触发重新拉取并更新缓存。后续可迭代为定时轮询。

**后端 API 端点**:
- `GET /api/analysis/summary` — 缓存状态 + 统计概览 KPI
- `GET /api/analysis/domains` — 按域名聚合的统计数据，支持 `type`/`search`/`sort`/`order`/`limit` 查询参数
- `GET /api/analysis/domains/:domain` — 该域名的上游明细 + 原始查询列表（含分页）
- `POST /api/analysis/refresh` — 触发后端重新从 AdGuardHome 拉取数据（异步）
- `GET /api/health` — 后端健康状态

**导出**:
前端生成 CSV，通过 Blob 下载。"导出统计"导出当前表格视图的全量数据；"导出日志"导出指定域名或全部域名的原始查询记录。

**外观主题**:
跟随系统 `prefers-color-scheme`，加手动 sun/moon 切换按钮。Tailwind `dark:` 变体 + shadcn/ui 主题系统。

**表格交互**:
Top N 域名表格支持列排序 + 模糊搜索（按域名关键词），可点击行内展开查看该域名的上游延时明细。不用 Modal 或抽屉。

**项目结构**:
npm workspaces monorepo：`packages/client/`（Vite + React SPA）、`packages/server/`（Fastify + TypeScript）、`shared/`（前后端共享 API 类型定义）。
